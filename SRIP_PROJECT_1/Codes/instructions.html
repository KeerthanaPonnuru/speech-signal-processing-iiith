Usage

If Tone.js is included in the page, a global variable named Tone will be added to the window. Module Loaders

Internally, Tone uses AMD for dependency management. But the build process adds a UMD footer to the built file which makes it compatible with both
AMD and CommonJS.


If you have XCode installed on your Mac, you should be able to get the examples running with the following steps: Download the .zip file from github: https://github.com/Tonejs/Tone.js/archive/dev.zip

The file should unzip automatically, so then in a Terminal window go into the directory:

$ cd Tone.js-dev

and run the commands:

$ npm install ... $ npm run build

For Aliasing and Quantization change the Tempo and the corresponding sampling

For Filter Design change the partial count and observe the corresponding change in waveform

The in built-audio for waveform and waveform enables you to view the waveform and pause and notice changes in pattern respectively
